# 音声再生画面 詳細設計書

## 要件整理

### 機能概要

タイムラインから録音をタップした際に、専用の再生画面を表示する。
一時音声確認画面のUIを流用する。

### 現状の課題

- タイムラインの「Listen Now」ボタンで再生が始まるが、専用画面がない
- 再生中の操作（一時停止、シーク、停止）ができない

### 実装方針

#### 影響するレイヤー

- **presentation層**：
  - 音声再生画面の新規作成
  - タイムラインからの遷移

- **application層**：
  - 既存の audio_player_provider を使用（変更なし）

- **domain層**：
  - 変更なし

- **infrastructure層**：
  - 変更なし

#### 新規作成ファイル

```
lib/presentation/
  └── screens/
      └── playback_screen.dart
```

#### 修正が必要な既存ファイル

- `lib/presentation/widgets/recording_card.dart`（画面遷移追加）
- `lib/presentation/screens/recording_screen.dart`（一時確認画面のウィジェット抽出）

#### 使用する依存パッケージ

- flutter_riverpod（既存）
- just_audio（既存）
- intl（既存）

#### 状態管理への影響

- 既存の `audioPlayerProvider` を使用
- 新規プロバイダーは不要

### 機能要件

#### 基本機能

1. **再生/一時停止**：中央の大きなボタン
2. **シークバー**：タップ＆ドラッグでシーク可能
3. **再生時間表示**：現在時間 / 総時間
4. **波形表示**：視覚的フィードバック（オプション）

#### 追加機能

1. **共有ボタン**：録音ファイルを共有
2. **削除ボタン**：録音を削除（確認ダイアログ表示）
3. **戻る/進むボタン**：10秒スキップ（オプション）

### コンポーネント設計

#### PlaybackScreen

- Recording オブジェクトを受け取る
- 既存の PlaybackControls ウィジェットを流用

#### 流用するウィジェット

- `lib/presentation/screens/recording/widgets/playback_controls.dart`
  - 再生/一時停止ボタン
  - シークバー
  - 時間表示

### 想定されるリスクとエッジケース

1. **ファイルが存在しない場合**：エラーメッセージ表示
2. **再生中に画面を閉じた場合**：再生を停止するか継続するか
3. **複数の録音を連続再生**：次の録音への自動遷移（オプション）
4. **バックグラウンド再生**：今回は対応しない

### 実装の流れ

1. **既存ウィジェットの抽出**
   - PlaybackControls を共通ウィジェットとして抽出

2. **PlaybackScreen の作成**
   - Recording を受け取る
   - PlaybackControls を組み込み
   - 共有・削除機能を追加

3. **タイムラインからの遷移**
   - RecordingCard の Listen Now ボタンで画面遷移

4. **削除機能の実装**
   - 確認ダイアログ
   - Repository 経由で削除
   - タイムラインに戻る

## 確定事項

1. 波形表示：必要（実装する）
2. 再生中に画面を閉じた場合：再生を停止しない（バックグラウンド再生を継続）
3. 10秒スキップボタン：必要（実装する）
4. 削除機能：必要（確認ダイアログ付きで実装する）

### 追加機能要件

- 波形表示を実装（just_audio のストリームから生成）
- 10秒戻る / 10秒進むボタンを再生/一時停止ボタンの左右に配置
- 削除ボタン：確認ダイアログ表示後、Repository経由で削除しタイムラインに戻る
- 画面を閉じても再生は継続する
