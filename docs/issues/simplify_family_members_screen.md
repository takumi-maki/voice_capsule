# Family Members画面のシンプル化と詳細画面の追加

## 背景

現在の Family Members 画面では、各カードに編集・削除ボタンが直接表示されている。
これをシンプルなカード一覧に整理し、削除操作は詳細画面に移す。

## 現状の問題

```
┌────────────────────────────────┐
│  Family Members                │
│                                │
│  ┌──────────────────────────┐  │
│  │ 👶 Takumi        ✏️  🗑  │  │  ← 編集・削除ボタンがカードに露出
│  └──────────────────────────┘  │
│                                │
│  + Add Child                   │
└────────────────────────────────┘
```

## 変更内容

### 1. Family Members 画面のシンプル化

**削除するUI要素:**
- カードの trailing にある編集アイコンボタン（`Icons.edit_outlined`）
- カードの trailing にある削除アイコンボタン（`Icons.delete_outline`）

**変更後のカード動作:**
- カードタップ → `ChildDetailScreen` へ遷移

**目指すUI:**

```
┌────────────────────────────────┐
│  Family Members                │
│                                │
│  ┌──────────────────────────┐  │
│  │ 👶 Takumi                │  │  ← シンプルなカード（ボタンなし）
│  └──────────────────────────┘  │
│                                │
│  + Add Child                   │
└────────────────────────────────┘
```

### 2. ChildDetailScreen の新規作成

カードタップ時に遷移する詳細画面を新規作成する。

**表示内容:**
- AppBar タイトル: 子どもの名前
- 子どものアバター（大サイズ）
- 名前
- 編集ボタン（`ChildProfileSetupScreen` へ遷移）
- 削除ボタン（確認ダイアログ → 削除 → FamilyScreen へ戻る）
  - 子どもが1人の場合は削除ボタンを非活性

**目指すUI:**

```
┌────────────────────────────────┐
│  ← Takumi                      │  ← AppBar（戻るボタン付き）
│                                │
│          （大アバター）          │
│          Takumi                 │
│                                │
│  ┌──────────────────────────┐  │
│  │        Edit Profile      │  │  ← アウトラインボタン
│  └──────────────────────────┘  │
│                                │
│  ┌──────────────────────────┐  │
│  │        Delete            │  │  ← エラーカラーのテキストボタン or ボタン
│  └──────────────────────────┘  │  （1人のみの場合は非活性）
└────────────────────────────────┘
```

## 使用する依存パッケージ

追加なし（既存パッケージのみ）

## リスク・エッジケース

- 子どもが1人のみの場合: 詳細画面の削除ボタンを非活性にする
- 削除後の画面遷移: `Navigator.pop` で FamilyScreen に戻り、一覧を再読み込みする
- 編集後の画面遷移: `ChildProfileSetupScreen` から戻った後、詳細画面の表示を更新する
